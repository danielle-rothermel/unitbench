# Workbench Evolution — Overall Plan

Status: living overview for evolving unitbench from a read-only viewer into
the single workbench app for the whetstone/dr-* tool family. Companion
execution brief: [`prompt.md`](prompt.md). Mirrors the structure of
`../whetstone-ai/docs/composable/overall.md`.

## Framing

- **Libraries ship inspection surfaces, the workbench composes them.**
  Each dr-* library owns the headless capability (typed records out) and
  an optional thin FastAPI facade (`[serve]` extra); unitbench owns all
  UI. Cross-cutting views only work if there is exactly one app.
- **Web is the prototyping environment.** No notebook tier. Disposable
  experiments live in a `/lab` route group (design-system-exempt,
  guilt-free deletion); promotion out of `/lab` means adopting the design
  system.
- **Lanes** (target IA): **Data** (browse/dashboard/metrics), **Replay**
  (trace/step-through/diff), **Playgrounds** (live tools hitting library
  facades), **Design** (graph viewer/builder, platform map), plus `/lab`.

## House rules

1. **Three-layer tools.** Library function returning typed records →
   optional FastAPI facade → thin UI. Logic never lives in the app.
2. **Typed facades, one pattern.** Pydantic models → FastAPI router in
   the library's `[serve]` extra → OpenAPI → generated TS client
   (`openapi-typescript`), regenerated by one command. The facades are
   the JSON contract.
3. **`/lab` disposability.** Prototype pages are exempt from design
   polish and deletable without ceremony; nothing outside `/lab` may
   import from it.
4. **Design system discipline.** Extend the OKLCH tokens, never bypass
   them; product register rules apply (density is a feature, one shared
   "inspect anything" pattern, charts themed from the tokens); light
   mode only; Fira Code for code/data; display font reserved for true
   display moments.
5. **Never touch the migration.** whetstone-ai and the library repos'
   `composable-migration` branches are read-only from here. Library-side
   facade work happens on separate branches, gated on the migration
   log showing the relevant stage done.
6. **Hostable vs local-first.** Retrospective lanes (data/replay) deploy
   to Vercel+Neon. Playground lanes are local-first; a facade may go to
   Railway only if it holds no API keys (compute-only) or gains auth.

## Stages

### Stage 0 — Correctness and polish fixes (the critique backlog)

Order: trust/correctness → reading ergonomics → typography → power layer.
Decisions already made: heatmap is **fixed in place** as a real matrix
(multi-kind data is coming); all fixes land before any restructuring.

1. **Diagnostics integrity** (P0, root-caused in
   `src/lib/prediction-diagnostics.ts`): `buildScoringStage` maps
   `scoring_status === 'scored'` → "Passed" — split stage *completion*
   (neutral "Scored — 0.00") from *outcome*; `buildEvaluationStage`
   renders `failures/total` unvalidated ("1578/1100 failed") — clamp the
   math and render an explicit inconsistent-data marker when
   `failures > total` (do not silently normalize). Note for later: check
   upstream unit semantics when v1 projections replace published tables.
2. **Heatmap rebuild in place** (P1×2): real `role="grid"`/table
   semantics with axis headers and accessible cell names ("model, kind:
   value"); contrast-computed cell text color (fixes measured 3.76:1);
   client-only mount gate for DndContext (fixes the hydration mismatch
   console error).
3. **Detail-page disclosure** (P2): five debug JSON payloads collapsed
   by default with line-count summaries (prompt/code/raw-generation stay
   open); deduplicate triple-rendered provenance.
4. **Typography/consistency pass**: rein Space Grotesk back to display
   moments (measured 19–31% of text); cap explainer prose ~72ch; align
   the filter grid; unify `unknown` vs `—`; replace ASCII `<-` glyph.
5. **Small power layer** (P3): `/` focuses filter, `j/k` row nav; wire
   or drop the "12 visible columns" label.

Every fix lands with a Playwright regression test (`e2e/`).

### Stage 1 — IA restructure + design system

Lane-based navigation (Data/Replay/Playgrounds/Design + `/lab`); extract
the component inventory into `DESIGN.md`; choose the chart library and
build the token-mapped chart theme (demo in `/lab`); build the shared
**inspector** component (structured payload + provenance + copy + code
panes) that replay/playgrounds/detail will reuse.

### Stage 2 — Facade pattern + parser playground

*Gate: whetstone migration log shows dr-code stage done.* Add the
`[serve]` extra to dr-code (own branch, not the migration branch) exposing
`explain(text, profile, stages=...)`; generate the TS client; build
`/playgrounds/parser`: paste text → toggle stages → candidate tree with
winner rationale. This is aspiration 5 and the most-pressing tool.

### Stage 3 — Provider query page + variance mode

*Gate: dr-providers v0.2 done.* `[serve]` extra over
generate/build_payload/conformance; `/playgrounds/provider`: request
builder → wire payload preview → response + conformance violations →
retry. Variance mode: prompt × model × N → JSONL download + variance
report (library function; shares machinery with canary probes).
Local-only (holds keys).

### Stage 4 — Graph viewer

*Gate: dr-graph exists.* `/design/graph`: render any GraphSpec JSON
(paste/upload), validated against the dr-graph JSON schema. Pure
frontend. Builder is deferred.

### Stage 5 — Projection read layer + dashboard lane

The keystone (aspirations 1–3). Co-design the projection schema as the
consumer sketch for the platform library's rebuildable-projections
feature; build the dashboard: correctness-vs-compression scatters,
heatmaps, violins — every plot point click-through to the sample
detail. Until v1 projections exist, build against the current
`published_*` tables behind a read-layer interface so the swap is
contained.

### Stage 6 — Replay viewer

Aspiration 4a: step-through of a generation run from `node_attempts` +
`graph_snapshot`, with side-by-side compare of sibling predictions.
Hardest UI; reuses the inspector component and graph viewer.

## Decisions made in flight

- **Chart library: visx** (stage 1). SVG-native so the OKLCH tokens
  pass straight through as `var(--…)` fill/stroke strings
  (`src/lib/chart-theme.ts`); tree-shakeable; low-level enough for the
  stage-5 violins. Demo scatter at `/lab/chart-demo`.
- **Lane IA keeps existing URLs** (stage 1). URL-addressable state is a
  praised feature; the lanes are navigation groups in `AppShell`
  (Data / Replay / Playgrounds / Design / Lab), with planned lanes
  shown as muted placeholders until stages 2–6 fill them. No redirects
  needed.
